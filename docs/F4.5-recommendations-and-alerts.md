## Overview

Build a recommendation engine that proactively surfaces actionable advice (rule-based + LLM-assisted) and an alert system with configurable thresholds. Alerts and recommendations are visible in the dashboard and available as context for the AI chat.

## User Stories

### US-1: Implement rule-based recommendations
**As a** homelab owner
**I want** the advisor to proactively tell me about problems and optimizations
**So that** I catch issues before they become outages

#### Acceptance Criteria
- At least 5 recommendation rules implemented:
  - High CPU on a Pi (>80% sustained) → suggest migrating workload to HOLYGRAIL
  - Disk usage >85% → warn about storage
  - Service down for >5 minutes → suggest investigation
  - Ollama not running → warn that AI features are degraded
  - New unknown device on network → security alert
- Rules evaluated on each data refresh cycle
- Recommendations displayed in a dashboard panel

### US-2: Configure alert thresholds
**As a** homelab owner
**I want** to set custom thresholds for CPU, memory, disk, and latency alerts
**So that** alerts match my tolerance levels and don't create noise

#### Acceptance Criteria
- Default thresholds set for common metrics (CPU, memory, disk, ping)
- Thresholds editable via the dashboard settings page
- Thresholds persisted to database
- Changes take effect on next evaluation cycle

### US-3: View alert history log
**As a** homelab owner
**I want** a searchable log of all past alerts and recommendations
**So that** I can review what happened and when

#### Acceptance Criteria
- Alert log page showing: timestamp, severity, device/service, message
- Filterable by severity (info, warning, critical), device, and date range
- Alerts marked as acknowledged/resolved
- Log retained for at least 30 days

### US-4: Push alerts to Home Assistant (optional)
**As a** homelab owner
**I want** critical alerts forwarded to Home Assistant as notifications
**So that** I get alerted on my phone even when not looking at the dashboard

#### Acceptance Criteria
- Optional integration: configurable HA webhook or REST API endpoint
- Only critical-severity alerts pushed (configurable)
- HA notification includes: alert message, affected device, timestamp
- Integration can be enabled/disabled without restarting the advisor
