## Overview

Evaluate the current NAS setup (Pi 4 + OpenMediaVault, 4.6TB) and plan a storage upgrade that supports growth beyond current capacity with improved performance and redundancy.

## User Stories

### US-1: Assess current storage capacity and performance
**As a** Camelot admin
**I want** a clear picture of current storage usage, growth rate, and performance
**So that** I can make an informed upgrade decision

#### Acceptance Criteria
- Current usage documented: total capacity, used, free, by media type
- Growth rate estimated (GB/month over last 6 months)
- Benchmark results from NAS (SMB read/write from Mac and Torrentbox)
- Pain points documented (speed, reliability, capacity pressure)

### US-2: Evaluate NAS upgrade options
**As a** Camelot admin
**I want** to compare upgrade paths (TrueNAS on HOLYGRAIL vs dedicated NAS hardware)
**So that** I pick the right solution for cost, performance, and reliability

#### Acceptance Criteria
- At least 2 options evaluated with pros/cons:
  - TrueNAS on HOLYGRAIL (second disk, ZFS)
  - Dedicated NAS appliance (Synology, QNAP, DIY)
  - Keep Pi 4 NAS with larger drives
- Cost estimates for each option
- Decision documented with rationale

### US-3: Plan zero-downtime migration
**As a** Camelot admin
**I want** a migration plan that keeps media accessible throughout the transition
**So that** Plex, Sonarr/Radarr, and downloads are never interrupted

#### Acceptance Criteria
- Step-by-step migration plan documented
- Rollback strategy defined
- Estimated downtime (target: zero for reads, minimal for writes)
- All mount points identified: Torrentbox, HOLYGRAIL, Mac

### US-4: Execute migration and update mount points
**As a** Camelot admin
**I want** all devices pointing at the new storage with the old NAS Pi repurposed
**So that** the network uses the upgraded storage seamlessly

#### Acceptance Criteria
- New storage online and serving SMB shares
- All devices remounted: Torrentbox, HOLYGRAIL, Mac
- Media playback verified in Plex
- Torrent downloads landing on new storage
- Pi 4 NAS either repurposed or decommissioned
- INFRASTRUCTURE.md updated
