## Overview

Build a service registry that knows what's running where — Docker containers on HOLYGRAIL (via socket), containers on Pis (via Portainer API or SSH), and standalone services. Display health status in a dashboard with green/yellow/red indicators.

## User Stories

### US-1: Detect Docker containers on HOLYGRAIL
**As a** Camelot admin
**I want** the advisor to list all running containers on HOLYGRAIL
**So that** I can see the state of local services without using Portainer

#### Acceptance Criteria
- Docker socket mounted into advisor backend container
- Container list shows: name, image, status, ports, uptime
- Auto-refreshes on a short interval (every 60 seconds)
- Stopped/exited containers shown separately

### US-2: Monitor service health via HTTP and port checks
**As a** Camelot admin
**I want** the advisor to probe service endpoints and report health
**So that** I know immediately when a service is degraded or down

#### Acceptance Criteria
- Health checks defined per service (HTTP 200 check, TCP port open, or custom)
- Status: green (healthy), yellow (slow/degraded), red (down/unreachable)
- Check interval configurable (default: 60 seconds)
- Health history stored for trend analysis

### US-3: Display service status dashboard
**As a** Camelot admin
**I want** a dashboard page showing all services with their health status
**So that** I get an at-a-glance view of the entire infrastructure

#### Acceptance Criteria
- Grid or table of services grouped by host
- Each service shows: name, host, port, status indicator, last check time
- Clicking a service shows health history and details
- Overall system health summary at the top (e.g., "12/14 services healthy")

### US-4: Track services across Pis (remote hosts)
**As a** Camelot admin
**I want** the advisor to also monitor services on Torrentbox, NAS, and other Pis
**So that** the dashboard covers the full network, not just HOLYGRAIL

#### Acceptance Criteria
- Remote service definitions configurable (host, port, check type)
- Torrentbox services monitored: Deluge, Sonarr, Radarr, Prowlarr, etc.
- NAS services monitored: SMB, Pi-hole
- Connection failures to remote hosts shown as host-level alerts
