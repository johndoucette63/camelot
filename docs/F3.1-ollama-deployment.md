## Overview

Deploy a local LLM inference server on HOLYGRAIL using Ollama with GPU passthrough. This provides the AI backbone for the Network Advisor (Phase 4) and general-purpose LLM access on the LAN.

## User Stories

### US-1: Deploy Ollama container with GPU passthrough
**As a** homelab owner
**I want** Ollama running in Docker with access to the RTX 2070 Super
**So that** LLM inference uses GPU acceleration instead of CPU

#### Acceptance Criteria
- Ollama Docker container deployed with NVIDIA runtime
- GPU visible inside the container (`nvidia-smi` works)
- Container set to restart on boot
- Ollama API listening on port 11434

### US-2: Pull and configure default LLM model
**As a** homelab owner
**I want** Llama 3.1 8B pulled and ready to serve
**So that** there's a capable default model within the 8GB VRAM budget

#### Acceptance Criteria
- `ollama pull llama3.1:8b` succeeds
- Model fits in VRAM (verified via `nvidia-smi` during inference)
- Model responds correctly to test prompts

### US-3: Expose OpenAI-compatible API on LAN
**As a** homelab owner
**I want** the Ollama API accessible from any device on 192.168.10.0/24
**So that** the Mac, Pis, and the Advisor app can all use it

#### Acceptance Criteria
- Ollama bound to `0.0.0.0:11434` (not just localhost)
- `curl http://holygrail:11434/v1/chat/completions` works from the Mac
- OpenAI-compatible endpoint tested with a Python client
- Firewall (if any) allows port 11434 on LAN

### US-4: Benchmark and document baseline performance
**As a** homelab owner
**I want** tokens/sec and latency numbers documented
**So that** I have a baseline to compare against future model or hardware changes

#### Acceptance Criteria
- Tokens/sec measured for Llama 3.1 8B (prompt eval + generation)
- Response latency measured for typical advisor-length prompts
- VRAM usage documented
- Results recorded in `docs/HOLYGRAIL-setup.md` or a benchmarks section
