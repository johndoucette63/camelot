## Overview

Once HOLYGRAIL is running Ubuntu, integrate it into the existing Camelot management tooling so it can be managed alongside the Pis from the Mac workstation.

## User Stories

### US-1: Set up SSH key authentication from Mac to HOLYGRAIL
**As a** homelab owner
**I want** passwordless SSH access from my Mac to HOLYGRAIL
**So that** scripts and manual sessions connect without prompts

#### Acceptance Criteria
- Mac's `~/.ssh/id_ed25519.pub` copied to HOLYGRAIL's `authorized_keys`
- `ssh john@holygrail` connects without a password
- Password authentication disabled in sshd_config

### US-2: Add HOLYGRAIL to ssh-config and management scripts
**As a** homelab owner
**I want** HOLYGRAIL included in `scripts/ssh-config`, `pi-status.sh`, and `pi-update.sh`
**So that** I can manage it with the same tools as the Pis

#### Acceptance Criteria
- `scripts/ssh-config` has a `Host holygrail` entry with correct IP and user
- `pi-status.sh` reports HOLYGRAIL status (uptime, disk, Docker, GPU temp)
- `pi-update.sh` supports HOLYGRAIL (apt update + Docker updates)
- GPU temperature and `nvidia-smi` summary included in status output

### US-3: Update INFRASTRUCTURE.md with final HOLYGRAIL details
**As a** homelab owner
**I want** `docs/INFRASTRUCTURE.md` updated with HOLYGRAIL's final IP, OS, and service list
**So that** the documentation reflects the live state of the network

#### Acceptance Criteria
- HOLYGRAIL IP changed from TBD to actual assigned IP
- OS listed as Ubuntu Server 24.04 LTS
- All deployed services listed with ports
- Mermaid diagram updated

### US-4: Create HOLYGRAIL setup guide
**As a** homelab owner
**I want** a step-by-step `docs/HOLYGRAIL-setup.md` capturing everything done in Phase 1
**So that** the build is reproducible if I ever need to rebuild from scratch

#### Acceptance Criteria
- Covers: Ubuntu install, NVIDIA drivers, Docker, Portainer, SSH, networking
- All commands documented and copy-pasteable
- Hardware-specific notes (BIOS settings, boot order) included
