## Overview

Automate routine torrent hygiene on the Torrentbox — remove malicious executables, handle stalled downloads, and evaluate adding Readarr for book management.

## User Stories

### US-1: Schedule automated exe cleanup
**As a** Camelot admin
**I want** a cron job running `deluge-monitor.py --remove-exe` on a schedule
**So that** malicious executables in torrents are automatically removed

#### Acceptance Criteria
- Cron job or systemd timer configured on Torrentbox
- Runs `deluge-monitor.py --remove-exe` at least daily
- Results logged to a file for review
- Notification if executables were found and removed

### US-2: Automate stalled torrent cleanup
**As a** Camelot admin
**I want** stalled torrents detected and handled automatically
**So that** dead downloads don't consume disk space indefinitely

#### Acceptance Criteria
- `deluge-monitor.py --stalled` integrated into scheduled runs
- Configurable stall threshold (e.g., no progress for 48 hours)
- Stalled torrents paused or removed based on configuration
- Report of actions taken logged

### US-3: Evaluate Readarr for books
**As a** media consumer
**I want** to decide whether Readarr should replace or supplement LazyLibrarian
**So that** book downloads are managed consistently with TV and movies

#### Acceptance Criteria
- Readarr tested alongside or instead of LazyLibrarian
- Feature comparison documented (library management, indexer support, metadata)
- Decision made: replace LazyLibrarian, keep both, or stick with LazyLibrarian
- If adopting: Readarr deployed and configured on Torrentbox
