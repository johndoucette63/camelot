## Overview

Set up the container runtime and GPU acceleration stack on HOLYGRAIL. This enables all downstream services (Plex transcoding, Ollama inference, monitoring) to run as Docker containers with GPU passthrough.

## User Stories

### US-1: Install NVIDIA drivers and CUDA toolkit
**As a** homelab owner
**I want** the RTX 2070 Super recognized and accelerated under Ubuntu
**So that** GPU workloads (transcoding, LLM inference) can use the hardware

#### Acceptance Criteria
- NVIDIA driver installed (latest stable from ubuntu/nvidia PPA)
- `nvidia-smi` shows the RTX 2070 Super with driver version and CUDA version
- No X server / desktop environment installed (headless)

### US-2: Install Docker Engine and Docker Compose
**As a** homelab owner
**I want** Docker installed from the official apt repository
**So that** all services run as containers with Compose orchestration

#### Acceptance Criteria
- Docker Engine installed from official Docker apt repo (not Snap)
- Docker Compose v2 available as `docker compose`
- `john` user added to `docker` group (no sudo required)
- Docker service enabled on boot

### US-3: Install NVIDIA Container Toolkit
**As a** homelab owner
**I want** Docker containers to access the GPU via NVIDIA runtime
**So that** services like Ollama and Plex can use hardware acceleration inside containers

#### Acceptance Criteria
- `nvidia-container-toolkit` installed
- Docker configured with `nvidia` runtime in `/etc/docker/daemon.json`
- `docker run --gpus all nvidia/cuda:12.x-base nvidia-smi` succeeds
- GPU passthrough verified inside a container

### US-4: Deploy Portainer for container management
**As a** homelab owner
**I want** a web UI for managing Docker containers on HOLYGRAIL
**So that** I can monitor and manage containers without SSH

#### Acceptance Criteria
- Portainer CE deployed as a Docker container
- Accessible at `https://holygrail:9443` from the Mac
- Admin account created
- Docker socket mounted for local container management
