## Overview

Build the core advisor experience: a chat interface where the user asks natural-language questions about their network and gets answers grounded in live device inventory, service health, and alert data — powered by a local LLM via Ollama.

## User Stories

### US-1: Chat UI with message history
**As a** Camelot admin
**I want** a chat interface in the advisor dashboard
**So that** I can ask questions and get conversational responses

#### Acceptance Criteria
- Chat panel accessible from the main dashboard (sidebar or dedicated page)
- Message history displayed in a scrollable thread
- User messages and advisor responses visually distinguished
- Chat history persisted per session (optionally across sessions)

### US-2: Connect chat to Ollama LLM backend
**As a** Camelot admin
**I want** my questions answered by the local Ollama model
**So that** nothing leaves my network and responses are fast

#### Acceptance Criteria
- Backend calls Ollama's OpenAI-compatible API (`/v1/chat/completions`)
- Streaming responses displayed token-by-token in the chat UI
- Model configurable (default: Llama 3.1 8B)
- Graceful fallback message if Ollama is unreachable

### US-3: Dynamic system prompt with live network context
**As a** Camelot admin
**I want** the AI to know the current state of my network when answering
**So that** responses are grounded in reality, not just generic knowledge

#### Acceptance Criteria
- System prompt assembled at query time from: device inventory, service health, recent alerts
- Context includes: device count, offline devices, unhealthy services, recent events
- Prompt structured so the LLM can reference specific devices and services by name
- Context size managed to fit within model's context window

### US-4: Answer common network questions accurately
**As a** Camelot admin
**I want** the advisor to handle questions like "Which Pi is most loaded?" or "Is my NAS reachable?"
**So that** I get actionable answers without manual investigation

#### Acceptance Criteria
- Tested against at least 8 representative questions:
  - "What devices are on my network?"
  - "Which services are down right now?"
  - "Is the Torrentbox VPN working?"
  - "What should I move from Pi to HOLYGRAIL?"
  - "Why might my Thread network be fragmented?"
  - "Summarize what's going on with alerts right now"
  - "NAS latency spiked at 2am — is that normal?"
  - "Anything unusual in the logs in the last 24 hours?"
- Responses reference real device names, IPs, and current status
- Monitoring-grounded questions pull context from InfluxDB/Grafana data and alert history
- Advisor admits uncertainty rather than hallucinating
