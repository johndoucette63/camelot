## Overview

Consolidate the monitoring stack (currently split across Pis) onto HOLYGRAIL and add a reverse proxy so all services get clean URLs on the LAN.

## User Stories

### US-1: Migrate monitoring stack to HOLYGRAIL
**As a** Camelot admin
**I want** Grafana, InfluxDB, and Smokeping running on HOLYGRAIL
**So that** monitoring is centralized on the most powerful machine

#### Acceptance Criteria
- Grafana, InfluxDB, and Smokeping deployed via Docker Compose on HOLYGRAIL
- Existing Grafana dashboards imported (network-monitoring.json)
- InfluxDB data migrated or fresh collection started
- Smokeping targets updated to reflect all current devices

### US-2: Deploy Traefik reverse proxy
**As a** Camelot admin
**I want** a reverse proxy giving clean URLs to all HOLYGRAIL services
**So that** I access services via names instead of port numbers

#### Acceptance Criteria
- Traefik deployed as a Docker container on HOLYGRAIL
- Services accessible via subdomains or paths (e.g., `plex.holygrail`, `grafana.holygrail`)
- Traefik dashboard accessible for route inspection
- DNS or `/etc/hosts` entries documented for Mac

### US-3: Verify monitoring continuity
**As a** Camelot admin
**I want** monitoring data flowing correctly after migration
**So that** there are no gaps in visibility

#### Acceptance Criteria
- Smokeping latency data collecting for all targets
- Grafana dashboards rendering with live data
- Speedtest logger running on schedule
- Wi-Fi monitor reporting if applicable

### US-4: Update infrastructure documentation
**As a** Camelot admin
**I want** docs and Docker Compose files to reflect the new service locations
**So that** the repo matches the live infrastructure

#### Acceptance Criteria
- `infrastructure/holygrail/docker-compose.yml` contains all HOLYGRAIL services
- `infrastructure/monitoring/` configs updated for HOLYGRAIL paths
- INFRASTRUCTURE.md service table updated
- Old Pi monitoring configs archived or removed
