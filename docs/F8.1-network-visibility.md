## Overview

Achieve full network observability: every device catalogued, all logs centralized, bandwidth tracked, and WiFi/DNS optimized. Produce a VLAN plan for future network segmentation.

## User Stories

### US-1: Complete device inventory on 192.168.10.0/24
**As a** homelab owner
**I want** every device on the LAN catalogued with role, owner, and network details
**So that** I have a definitive source of truth for the network

#### Acceptance Criteria
- Full nmap scan of 192.168.10.0/24 with OS detection
- Every device assigned a role in the advisor inventory
- Unknown or unexpected devices flagged
- Inventory exportable (CSV or JSON)

### US-2: Deploy centralized logging (Loki + Promtail)
**As a** homelab owner
**I want** logs from all Pis and HOLYGRAIL searchable in one place
**So that** I can troubleshoot issues across the network without SSH-ing into each device

#### Acceptance Criteria
- Loki deployed on HOLYGRAIL (Docker)
- Promtail agents deployed on Torrentbox, NAS, and HOLYGRAIL
- Logs searchable via Grafana (Loki data source)
- Docker container logs and syslog captured

### US-3: Monitor per-device bandwidth
**As a** homelab owner
**I want** to see network traffic per device over time
**So that** I can identify bandwidth hogs and diagnose slowdowns

#### Acceptance Criteria
- Bandwidth data collected (SNMP, iftop, or router stats)
- Per-device traffic graphed in Grafana
- Historical data retained for at least 30 days
- Alerts for unusual spikes configurable

### US-4: Optimize WiFi and DNS, plan VLANs
**As a** homelab owner
**I want** WiFi channels analyzed, DNS consolidated, and a VLAN plan documented
**So that** the network is optimized and ready for future segmentation

#### Acceptance Criteria
- WiFi channel scan performed; recommendations documented
- Pi-hole vs AdGuard Home evaluated; consolidation decision made
- VLAN plan documented: IoT, media, management, guest segments
- VLAN plan includes required router/switch configuration (implementation optional)
