## Overview

Move Plex from the Media Server Pi (192.168.10.150) to HOLYGRAIL, unlocking hardware transcoding via the RTX 2070 Super's NVENC encoder. Decide whether to keep Emby as a fallback or retire it.

## User Stories

### US-1: Deploy Plex on HOLYGRAIL with GPU transcoding
**As a** media consumer
**I want** Plex running on HOLYGRAIL with NVIDIA hardware transcoding
**So that** streams transcode quickly without CPU bottleneck

#### Acceptance Criteria
- Plex Docker container deployed with `--gpus all` or NVIDIA runtime
- NVENC hardware transcoding verified in Plex dashboard (Settings → Transcoder)
- Plex accessible at `http://holygrail:32400/web`
- Container set to restart on boot

### US-2: Migrate Plex libraries and metadata
**As a** media consumer
**I want** my existing Plex libraries, watch history, and metadata preserved
**So that** the migration is seamless with no re-scanning or lost progress

#### Acceptance Criteria
- NAS SMB shares mounted on HOLYGRAIL (`/mnt/nas/movies`, `/mnt/nas/tv`, etc.)
- Plex libraries pointed at NAS mount paths
- Plex metadata/database migrated from Pi or rebuilt via scan
- All existing media detected and playable

### US-3: Decide on Emby (keep or retire)
**As a** Camelot admin
**I want** to make a deliberate decision about Emby's future
**So that** I'm not running redundant media servers

#### Acceptance Criteria
- Decision documented in INFRASTRUCTURE.md (keep as fallback or retire)
- If keeping: Emby deployed on HOLYGRAIL alongside Plex
- If retiring: Emby container stopped on Pi, noted as decommissioned

### US-4: Repurpose or decommission Media Server Pi
**As a** Camelot admin
**I want** the Media Server Pi (192.168.10.150) freed up after Plex moves
**So that** the hardware is available for a new role or can be shelved

#### Acceptance Criteria
- Plex and Emby containers stopped on Pi
- Pi either assigned a new role or powered down
- INFRASTRUCTURE.md updated to reflect new Pi status
- `pi-status.sh` updated if Pi is removed from inventory
